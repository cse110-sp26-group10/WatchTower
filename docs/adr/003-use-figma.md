# Use Figma for Wireframes and Design

## Context and Problem Statement

The UI/UX of a software is a core part that can make or break a user's experience. Therefore, it makes sense to design and draft the frontend design of the software before starting on any HTML. This way, the team can visualize the software during the process of its creation. Additionally, the creation of the page also helps the team understand the flow and architecture of the software.


## Considered Options

* Google Slides
* Figma
* Miro

## Decision Outcome

Chosen option: **Figma**, because it is a powerful and easy to learn tool that lets us create our pages. It also comes with an education plan, which unlocks premium features without the team needing to pay.

### Consequences

* Good, because Figma has all the tools needed in one place to draft wireframes and the front end
* Good, because Figma allows real time collaboration between team members
* Bad, because there is a lack of solid source control
* Bad, because an external source of truth is introduced, other than just this repository
